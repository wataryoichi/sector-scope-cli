# /start-dev

このスキルは、このプロジェクトで実装作業を始める前の標準起動手順です。

## 目的
- 仕様を見落とさずに作業開始する
- TODO とハンドオフを確認して、現在地を把握する
- いきなり編集せず、まず理解を共有する

## 実行手順

1. まず `docs/spec.md` を読む。
2. `TODO.md` が存在すれば読む。
3. `docs/handoff-current.md` が存在すれば読む。
4. 主要ファイル・主要ディレクトリを確認する。
   - `README.md`
   - `pyproject.toml`
   - `src/`
   - `tests/`
   - `docs/`
5. ここまでの理解を、以下の形式で 5〜10 行程度に要約してから次へ進む。

```md
## 開始時点の理解
- プロジェクトの目的:
- 現在の実装状況:
- 未完了の主要タスク:
- 今回着手する内容:
- 影響しそうなファイル:
```

6. その後、今回の作業の最小実行プランを提示する。
7. ここまで終わったら実装を開始する。

## 重要ルール
- `docs/spec.md` を読まずに実装を始めないこと。
- 仕様と既存コードが矛盾したら、勝手に決め打ちせず、まず状況を短く説明すること。
- 大きめの変更では、実装前に対象ファイルを明示すること。
- 実装後は `TODO.md` / `docs/handoff-current.md` / 必要なら `docs/spec.md` を更新すること。

## よくある使い方
- `/start-dev`
- `/start-dev まず価格取得周りを把握してから実装開始`
- `/start-dev TODO の未完了タスクから優先順に進める`

## 補足
- このスキルは「開始フロー」です。実装方針相談は通常の対話でよいです。
- より細かい使い方は `/help-project` を案内してください。
